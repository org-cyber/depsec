depsec — dependency security gate

Intercepts package installations before they touch your filesystem. Validates names against typosquats. Scans the full dependency tree for malicious code. Verifies every checksum.

Installation

$ go install github.com/org-cyber/depsec/cmd/depsec@latest

# Or build from source:
$ git clone https://github.com/org-cyber/depsec.git
$ cd depsec
$ go build ./cmd/depsec

Alias your package manager for transparent interception:

# Add to ~/.zshrc or ~/.bashrc
alias npm="depsec npm"
alias pip="depsec pip"
alias cargo="depsec cargo"

Usage

$ depsec install npm express
[depsec] ALLOWED: express@5.2.1
[depsec] Scanned 45 unique packages in tree. Clean.
[depsec] Proceeding with install.

$ depsec install npm this-is-fake
[depsec] BLOCKED: package "this-is-fake" not found in registry
           Did you mean: this-is-real?

$ depsec scan package-lock.json

Behavior

Gate checks (before any download):

  • Registry existence — 404 blocks immediately. Catches AI-hallucinated package names before an attacker can register them.
  • Typosquat detection — Levenshtein distance + homoglyph normalization (Cyrillic е vs Latin e) against top-10,000 packages.
  • Project context — Flags new dependencies in mature projects. A package not in your existing package.json gets stricter scrutiny.

Scanner checks (after quarantine download):

  • Install script analysis — preinstall, postinstall, prepare containing network calls (curl, fetch, wget, node -e).
  • Persistence path detection — File writes to .claude/, .vscode/, .github/workflows/, or shell profiles.
  • Full tree coverage — Not just the package you asked for. Every sub-dependency is resolved, downloaded, and scanned concurrently.

Checksum verification — Every tarball is verified against the registry's declared hash before extraction. Prevents CDN compromise or MITM substitution.

Scanner Rules

Rule ID Severity Description
npm-postinstall-network Critical Install script (preinstall, postinstall, prepare) contains a network call or code execution.
npm-persistence-path Critical Package contains files targeting IDE/AI configuration directories (.claude/, .vscode/) or GitHub Actions workflows.

Why

Existing tools (Snyk, Dependabot, OSV) focus on known CVEs — version bumps for disclosed vulnerabilities. They don't catch valid packages with valid provenance that contain novel malware, typosquats with zero CVEs, or install-time persistence mechanisms.

depsec was built in response to the Mini Shai-Hulud supply chain campaign (May 2026) and the ongoing wave of Next.js ecosystem attacks. Deterministic security tools for deterministic threats.
