01 — The problem

What existing tools miss

Snyk, Dependabot, and OSV scan for known CVEs — version bumps for disclosed vulnerabilities. They don't catch what's actually attacking your machine today.

01
Typosquats & AI-hallucinated names
LLM coding assistants suggest package names that don't exist. Attackers register them and wait. One homoglyph off from lodash and arbitrary code runs on your machine.
02
Install-time persistence
The Mini Shai-Hulud campaign used postinstall scripts to write backdoors to .claude/ and .vscode/ — surviving npm uninstall entirely.
03
Zero-CVE novel malware
A freshly-published package with valid provenance and zero CVE history passes every existing scanner. depsec scans behavior, not reputation.
02 — Architecture

Two-layer defense,
before code runs

depsec resolves what would be installed, validates every name, downloads every tarball to a quarantined temp directory, and only then allows the real install.

01

Registry existence check

Queries the live npm, PyPI, or crates.io registry. If the package doesn't exist, blocks immediately and suggests the closest popular match.

02

Typosquat & homoglyph detection

Levenshtein distance against the top-10,000 packages, plus Cyrillic/Latin homoglyph normalization. A new package one edit away from lodash gets blocked.

03

Quarantine extraction + checksum

Every tarball in the full dependency tree is downloaded to a temp directory and verified against the registry's declared SHA1/SHA256. Never touches node_modules.

04

Concurrent rule-engine scan

A worker pool scans extracted contents for postinstall network calls, persistence path writes, and obfuscated payloads. Blocks the entire install on any Critical hit.

Deterministic rules,
auditable blocks

04 — Get started

Up in two minutes

Alias your package manager and depsec becomes invisible — you type npm install and depsec intercepts automatically.

Build from source01
# requires Go 1.22+ $ git clone github.com/you/depsec $ cd depsec $ go build ./cmd/depsec
Alias your package manager02
# .bashrc / .zshrc alias npm="depsec npm" alias pip="depsec pip" alias cargo="depsec cargo"
Intercept mode03
# transparent — works like npm $ npm install express $ npm install --save-dev jest $ npm install react@18
Scan existing lockfile04
$ depsec scan package-lock.json $ depsec scan requirements.txt $ depsec update-db
