// The problem

The supply chain is
under attack

Most tools scan your lockfile after install. By then, 50+ sub-dependencies have already executed arbitrary code on your machine.

Typosquats & hallucinated packages
AI coding assistants suggest package names that don't exist. Attackers register those names and wait. One character off from lodash and you're compromised.
Install-time malware
Packages like the Mini Shai-Hulud campaign run postinstall scripts that persist backdoors in .claude/ and .vscode/ — and survive npm uninstall.
Zero-CVE novel malware
Snyk, Dependabot, and OSV scan for known vulnerabilities. They won't catch a freshly-published malicious package with valid provenance and zero CVE history.
// Two-layer defense

Intercept before
code runs

Layer 01 — Fast Gate

Validate before
any download

Before a single tarball touches disk, depsec queries the live registry, runs typosquat detection using Levenshtein distance and homoglyph normalization, and checks project context.

  • Registry existence check against npm, PyPI, crates.io
  • Levenshtein + homoglyph detection vs. top-10k packages
  • Cyrillic/Latin lookalike normalization
  • Project context scoring for new dependencies

Layer 02 — Scanner

Scan every tarball
in quarantine

Packages that pass the gate are downloaded to a quarantined temp directory. Every package in the full dependency tree — not just top-level — is extracted, checksummed, and scanned by a concurrent rule engine.

  • SHA1/SHA256 checksum vs. registry-declared hash
  • postinstall scripts with network calls
  • Writes to .claude/, .vscode/, ~/.ssh/
  • Obfuscated payloads and high-entropy strings
  • Full tree scan — depth-3 malware still caught

// Detection rules

Current ruleset

Deterministic Go code — no LLM hallucinations can accidentally allow malware through. Every block is auditable.

Rule ID Severity What it detects
npm-postinstall-network Critical pre/postinstall scripts containing http, curl, wget, fetch, or node -e
npm-persistence-path Critical Files in tarball targeting .claude/, .vscode/, or .github/workflows/
npm-obfuscation Coming eval(atob(...)), high-entropy string detection
py-import-exec Coming __init__.py with network calls at module load time
cargo-build-rs Coming build.rs with network activity or file writes outside target/

// Get started

Install in two minutes

Requires Go 1.22+. Alias your package manager and depsec becomes invisible — you type npm install and depsec intercepts automatically.

Build from source 1
$ git clone github.com/you/depsec $ cd depsec $ go build ./cmd/depsec
Alias your package manager 2
# Add to .bashrc / .zshrc alias npm="depsec npm" alias pip="depsec pip" alias cargo="depsec cargo"
Intercept mode 3
# Transparent — works like npm $ npm install express $ npm install --save-dev jest
Scan existing lockfile 4
$ depsec scan package-lock.json $ depsec scan requirements.txt $ depsec update-db
